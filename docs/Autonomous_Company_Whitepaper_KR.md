## 1. 서론: 탄생 배경과 필요성

### 1.1. 탄생 배경
초거대 인공지능(LLM)과 자동화 스크립트 기술이 비약적으로 발전함에 따라, 인간의 노동을 보조하는 수준을 넘어 **'기업의 운영 자체를 소프트웨어가 전담할 수 있는가?'**라는 질문이 대두되었습니다. 기존의 퀀트 트레이딩 봇이나 단순 업무 자동화(RPA)는 한정된 규칙 내에서만 작동할 뿐, 예기치 않은 오류나 시장의 급변, 규제 변화 앞에서는 결국 인간의 개입이 필수적이었습니다. 
AutonomousCompanyMVP는 이러한 한계를 극복하고, 스스로 수익을 창출하면서도 컴플라이언스와 리스크를 스스로 통제하는 **'진정한 의미의 무인 기업'**을 실현하기 위한 첫 번째 프로토타입으로 탄생했습니다.

### 1.2. 필요성 및 추진 배경
- **24/7/365 무중단 시장 대응:** 글로벌 크립토/금융 시장은 쉬지 않고 돌아갑니다. 인간의 수면이나 휴식 시간으로 인한 운영 공백은 치명적인 리스크입니다.
- **휴먼 에러(Human Error)의 원천 차단:** 피로도나 감정에 의한 오주문(Fat-finger), 룰 위반을 배제하고 냉철한 기계적 규율(Discipline)을 강제할 시스템이 필요합니다.
- **컴플라이언스와 감사(Audit) 비용 폭증:** 갈수록 복잡해지는 금융 규제 속에서, 모든 의사결정의 계보(Lineage)를 완벽하게 기록하고 증명하는 체계의 자동화가 시급해졌습니다.

## 2. 무인회사의 개념과 목표

### 2.1. 개념 (Concept)
무인회사(Autonomous Company)란 **"수익 창출, 리스크 관리, 규제 준수, 장애 복구 등 기업의 필수 생존 활동을 다수의 AI 에이전트와 알고리즘이 협력하여 인간의 지속적인 개입 없이 자율적으로 수행하는 소프트웨어 기반 법인"**입니다. 
현재 MVP 모델은 가장 직접적이고 정량적인 수익 창출원인 **'자동 매매(Automated Trading)'**를 핵심 수익 엔진으로 삼고 있습니다.

### 2.2. 목표 (Goals)
1. **안전 최우선 (Fail-Closed Safety):** 자율성이 시스템을 파괴하지 않도록, 확신이 서지 않는 상황에서는 즉시 거래를 중단하고 자산을 보호하는 'Fail-Closed'를 최우선으로 합니다.
2. **투명한 의사결정 증명:** 모든 AI의 판단과 알고리즘의 주문 내역은 왜(Why), 어떤 데이터(Feature)를 기반으로, 어떤 모델(Version)이 결정했는지 100% 추적 가능해야 합니다.
3. **점진적 자율화 (Phased Autonomy):** 처음부터 완전 자율을 추구하지 않습니다. 인간의 감독(Supervised) -> 제한적 자율(Limited) -> 완전 자율(Full Autonomy)로 이어지는 게이트를 스스로 통과해야만 권한이 확대됩니다.

## 3. 기존 시스템과의 차별성

단순한 트레이딩 봇이나 LangChain 기반의 토이 프로젝트와 AutonomousCompanyMVP는 근본적인 궤를 달리합니다.

1. **오케스트레이션과 권한 분리 (Separation of Duties):**
   - 수익을 내려는 에이전트(Revenue), 리스크를 막으려는 에이전트(CRO), 운영 안정성을 챙기는 에이전트(COO)가 분리되어 서로 견제합니다.
2. **인간 승인 정책의 코드화 (Policy Enforcer):**
   - AI가 스스로 리스크 한도를 마음대로 늘릴 수 없도록, 고위험 변경 시 `human_approval.json` 같은 명시적 서면 결재(암호화된 증적)를 요구합니다.
3. **AI 다중 교차 검증 (Cross-Verify):**
   - 단일 LLM의 환각(Hallucination)이나 편향을 막기 위해, 시스템 주요 설정 변경 전 반드시 3개 페르소나가 회의를 거쳐 합의점(Consensus)과 충돌점(Conflicts)을 리포팅합니다.
4. **WORM(Write Once Read Many) 방식의 감사 로그:**
   - 모든 체결(Fill) 정보에 해시값과 의사결정 ID를 매핑하여 사후 조작이 불가능한 형태의 감사를 지원합니다.

## 4. 작동 원리 및 아키텍처 구조

무인 회사의 두뇌는 노벨 경제학상 수상자 대니얼 카너먼의 인간 사고 체계(System 1 & System 2)를 모방하여 설계되었습니다.

### 4.1. 구조적 레이어 (Architecture Layers)
* **System 1 (고속 실행 루프):** `src/system1/executor.py`
  - 시장의 틱(Tick) 데이터를 밀리초(ms) 단위로 수집하고, 즉각적인 매수/매도/관망(Hold) 신호를 발생시킵니다.
* **System 2 (저속 계획 및 정책 루프):** `src/system2/planner.py`
  - 일정 주기마다(또는 시장 변동성 폭발 시) 거시적인 시장 데이터를 분석하고, AI 에이전트를 호출하여 System 1이 따를 '가이드라인(Policy)'을 업데이트합니다.
* **통제 엔진 (Control Engine):** `src/risk/guard.py`
  - System 1이 아무리 매수를 외쳐도, 통제 엔진(RiskGuard)에서 "일일 손실 한도 초과" 또는 "변동성 브레이커"를 발동하면 주문은 라우터(OrderRouter)에서 즉각 차단됩니다.
* **MLOps / 데이터 파이프라인:**
  - 데이터를 수집하고, 결측치를 정제하며, 무결성 해시를 부여해 레지스트리에 저장합니다. 데이터가 오염되면(Drift) 시스템은 스스로 보호 모드에 들어갑니다.

### 4.2. 자율 판단 프로세스 흐름
1. **데이터 파이프라인 실행** -> 2. **시장 예측 및 모델 런타임** -> 3. **Risk Guard의 정책 검사** -> 4. **주문 라우팅(Order Router)** -> 5. **체결 및 감사(Audit) 기록** -> 6. **사후 정합성(Reconcile) 및 게이트 평가**

## 5. 핵심 기능 및 특징 (현재 수행 가능한 업무)

현재 MVP 코드베이스 내에서 무인회사는 다음의 업무들을 "사람의 개입 없이" 수행할 수 있습니다.

| 기능명 | 설명 및 특징 | 작동 모듈 |
|---|---|---|
| **100대 리스크 자동 방어** | 극단적 손실, 대량 주문, 시장 붕괴 등 100가지 한계 상황에서 자산을 지켜내는 하드코어 방어막 | `RiskGuard`, `OrderRouter` |
| **자가 치유 (AutoHealer)** | 거래소 점검(503), API 호출 한도 초과(429) 등 인프라 장애 발생 시 스스로 대기시간(Backoff)을 계산하여 재시도 및 슬립(Sleep) 처리 | `auto_healer.py` |
| **자율 목표 수행 (OpenCLO)** | "게이트 실패 원인을 찾아 고쳐줘"라는 텍스트 명령만 주면, 에이전트가 알아서 여러 진단/복구 스크립트를 조합해 실행하고 결과를 보고 | `run_openclo_pilot.py` |
| **다중 AI 교차 검증 회의** | 새로운 시스템 설정이 제안되면 CRO(리스크), COO(운영), Revenue(수익) 3명의 AI가 서로 토론하고 신뢰도(Confidence) 점수를 산출 | `cross_verify.py` |
| **Sovereign Ops 웹 관제** | 인간 감독관을 위해 시스템 상태, 리스크 지표, 게이트 통과 여부를 실시간 글래스모피즘(Glassmorphism) UI로 시각화 (Prometheus 메트릭 포함) | `monitoring_server.py`, `web/monitoring` |
| **운영 챗봇 어시스턴트** | "현재 가장 위험한 요소 3가지를 요약해"라고 물으면, 대시보드의 실시간 데이터를 읽어와서 한국어로 브리핑 및 조치 플랜 제공 | `Gemini API` + `/api/chat` |
| **계좌 및 API 관리** | 마이페이지(My Page)를 통해 케이뱅크 계좌번호, 업비트 및 바이낸스 API 키를 안전하게 등록하고 로컬 저장소에 보관 | `web/monitoring` (localStorage) |
| **컴플라이언스 강제** | 승인되지 않은 실거래 자본 투입 시, `policy_enforcer.py`가 이를 적발하고 주문 차단 및 알림(Slack/Discord) 발생 | `policy_enforcer.py` |

## 6. 무인회사의 미래와 확장성

현재의 AutonomousCompanyMVP는 "가상/소액 실거래를 통한 안전성 검증" 단계에 머물러 있지만, 이 견고한 아키텍처를 바탕으로 다음과 같은 미래 비전을 실현할 수 있습니다.

### 6.1. 단기 진화 (1~2년 내)
- **자본의 동적 배분 (Capital Allocator AI):** 단일 전략을 넘어 수십 개의 트레이딩 알고리즘을 동시 가동하고, 성과가 좋은 알고리즘으로 회사의 자본을 실시간으로 자동 재배분합니다.
- **다중 거래소 및 자산 차익거래:** 바이낸스뿐만 아니라 주식, 외환, 탈중앙화 거래소(DEX)까지 연동하여 시장 간의 비효율을 포착하는 기관급 시스템으로 격상됩니다.
- **n8n / LangGraph 전면 도입:** 복잡한 에이전트의 워크플로우를 시각화하여, 장애 발생 시 AI가 슬랙으로 보고서를 보내고 즉각적인 슬랙 버튼 클릭만으로 복구를 승인받는 체계가 완성됩니다.

### 6.2. 중기 진화 (3~5년 내)
- **B2B 백오피스 완전 자동화:** 세금 신고 기간이 되면 시스템이 1년간의 체결 내역을 모아 국가별 세법에 맞춰 FIFO/LIFO로 자본이득세를 자동 계산하고, 국세청 API나 세무 법인에게 리포트를 자동 전송합니다.
- **법무 및 컴플라이언스 봇:** 새로운 규제(예: MiCA 등)가 발표되면 AI가 규제 원문을 스크래핑하여 내부의 `operational_limits.json`을 규제에 맞게 스스로 패치(Patch)안을 상정합니다.

### 6.3. 장기 비전: DAO를 넘어선 진정한 '자율형 주식회사'
- **스스로를 고용하고 해고하는 회사:** 시스템이 특정 영역의 로직 개발이 필요하다고 판단하면, GitHub/Upwork 등에 바운티(현상금)를 걸어 외부 프리랜서나 또 다른 AI에게 코딩을 외주 주고, PR을 자동 리뷰하여 병합(Merge)합니다.
- **결론적으로 무인회사는**, 단순한 트레이딩 계좌가 아니라, 스스로 자본을 굴리고, 인프라 비용을 지불하며, 국가에 세금을 내고, 생태계를 확장하는 **'디지털 생명체이자 하나의 완전한 법인(Corporation)'**으로 거듭나게 됩니다.

## 7. 기술적 심화: 100대 시나리오 검증 결과

무인 회사의 생존 능력을 검증하기 위해, 시스템의 5대 핵심 모듈(리스크 제어, 주문 체결, 자가 치유, AI 요원 판단, 규제 준수)을 대상으로 총 100가지의 극한 상황(Edge Cases) 매트릭스 테스트를 수행했습니다.

### 7.1. 테스트 카테고리 및 결과 요약
*   **리스크 제어 (Risk Guard):** 일손실 폭락(-10%), 시장 변동성 폭주(12%) 등 20개 시나리오에서 100% 자산 보호(Hold/Kill-switch) 확인.
*   **주문 체결 (Execution):** 한도 초과 주문($30,000 이상), 비정상 수량 등 20개 시나리오에서 라우터 차단 및 정상 체결 성공 확인.
*   **자가 치유 (AutoHealer):** API 429(Rate limit) 연속 발생 시 지수 백오프 작동, 503 유지보수 시 안전 셧다운 등 20개 복구 시나리오 통과.
*   **AI 요원 진단 (Agentic):** 복합 장애 상황에서 AI가 우선순위(P0/P1) 조치 계획을 올바르게 수립하는지 20개 시나리오 검증.
*   **규제 준수 (Policy):** 인간 승인 파일 없이 설정값을 무단 변경하려는 시도에 대한 20개 차단 시나리오 통과.

**최종 검증 결과:** 통과율 **100% (100/100 PASS)**. 
상세 리포트는 `outputs/100_scenarios_report.md`에 보관되어 있으며, 이는 무인 회사가 실제 시장의 불확실성 속에서도 원칙을 지키며 생존할 수 있음을 기술적으로 증명합니다.

## 8. 결론

**AutonomousCompanyMVP**는 AI가 인간을 대신해 돈을 버는 것을 넘어, **'돈을 버는 과정에서 파생되는 모든 기업적/운영적 책임(리스크, 보안, 규제, 복구)을 어떻게 소프트웨어로 캡슐화할 것인가'**에 대한 가장 실증적인 해답입니다.

코드베이스에 적용된 100여 가지의 리스크 차단 시나리오 검증, 무결성 해시 로깅, AI 교차 검증은 "안전이 담보되지 않은 자율성은 재앙"이라는 철학을 증명하고 있습니다. 우리는 이 MVP를 통해 인류가 곧 맞이할 **'노동자 없는 1인 AI 유니콘 기업'**의 확실한 청사진을 완성했습니다.
